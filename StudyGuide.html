    • Pulling changes from a remote repository
    Version Control Study Guide

    Version Control

    • Understand what the basic work flow is (stage, commit, push, pull)

      The basic workflow in version control involves:

      • Staging changes
      • Committing changes
      • Pushing changes to a remote repository
      • Pulling changes from a remote repository

    • Understand basic git commands (including checkout, clone, status, diff, merge, branch, and the ones above)

      Basic Git commands include:

      • checkout
      • clone
      • status
      • diff
      • merge
      • branch
      • and the ones mentioned above

    • Understand what .gitignore does

      The .gitignore file specifies intentionally untracked files that Git should ignore. It's useful for excluding files from version control that are generated automatically or contain sensitive information.

    • Be able to solve basic git problems (merge conflicts)

      Merge conflicts occur when Git is unable to automatically merge changes from different branches. To solve merge conflicts, one needs to manually resolve the differences between the conflicting changes and then commit the resolved changes.
