Design Patterns Study Guide

  • Understand the format/use cases of the following design patterns:
    • Model-View-Controller (MVC): Separates the application logic into three interconnected components: Model, View, and Controller.
    • Adapter: Allows incompatible interfaces to work together. It acts as a bridge between two incompatible interfaces.
    • Bridge: Separates an objectâ€™s abstraction from its implementation so that they can be modified independently.
    • Builder: Constructs complex objects step by step. It allows the construction of different types and representations of an object using the same construction code.
    • Decorator: Allows behavior to be added to individual objects dynamically without affecting the behavior of other objects from the same class.
    • Singleton: Ensures that a class has only one instance and provides a global point of access to that instance.
    • Factory: Creates objects without specifying the exact class of object that will be created.
    • Lazy Initialization: Delays the creation of an object or the calculation of a value until the first time it is needed.
    • Iterator: Provides a way to access the elements of an aggregate object sequentially without exposing its underlying representation.
    • Null Object: Provides a way to handle null references transparently by providing a substitute for the null object.
    • Object Pool: Reuses a set of objects rather than creating and destroying them as needed, improving performance and reducing memory overhead.
    • Proxy: Provides a surrogate or placeholder object to control access to another object.
    • Template Method: Defines the skeleton of an algorithm in the superclass but lets subclasses override specific steps of the algorithm without changing its structure.
